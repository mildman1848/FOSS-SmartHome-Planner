# Contributing

English: [CONTRIBUTING.md](CONTRIBUTING.md)

## Vor Einem Pull Request

- Änderungen fokussiert und gut beschrieben halten.
- Dokumentation und Changelog bei Bedarf aktualisieren.
- Vor dem Push die projektspezifischen Validierungsschritte ausführen.
- `docker compose` statt `docker-compose` verwenden.
- `hadolint.yml` als einzigen Dockerfile-Lint-Workflow beibehalten.
